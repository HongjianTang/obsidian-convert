## 目标

以最少的规则，保证：**正确性、可维护性、可扩展性、可观测性**。

---

## 质量维度与准则

### 正确性（Correctness）

转换工具的核心目标是**准确转换**，确保语义不丢失、链接不断裂。

- **转换完整性**：所有 Obsidian 语法正确转换，无遗漏
- **链接一致性**：转换后的链接指向正确目标
- **元数据保留**：重要的 frontmatter 字段正确映射
- **边界处理**：正确处理空文件、特殊字符、编码问题

**验收标准**：
- WikiLink 转换后链接可点击且指向正确文件
- 嵌入的图片在输出中正确显示
- Frontmatter 中的关键字段保留

### 可维护性（Maintainability）

代码应易于理解、修改和扩展。

- **分层清晰**：严格遵守依赖方向，Domain → Application → Presentation
- **职责单一**：每个类/函数只做一件事
- **命名清晰**：使用业务语言命名，避免技术术语污染领域模型
- **避免重复**：重复两次以上的代码应抽象

**代码评审重点**：
- 是否有跨层依赖？
- 领域逻辑是否泄漏到展示层？
- 转换器是否可独立测试？

### 可扩展性（Extensibility）

支持新的输入格式、输出格式和转换规则。

- **插件式转换器**：新增转换器无需修改核心代码
- **配置驱动**：转换行为可通过配置定制
- **接口抽象**：依赖接口而非实现

**扩展点**：
- 新增转换器（实现 `Transformer` 接口）
- 支持新的输入格式（新增 Parser）
- 支持新的输出格式（新增 Formatter）

### 可观测性（Observability）

问题可追踪、可定位。

- **结构化日志**：关键操作记录详细信息
- **错误可追踪**：错误信息包含文件名、行号、上下文
- **进度反馈**：长时间操作显示进度
- **调试模式**：`--verbose` 输出详细过程

## 转换质量标准

### 链接转换

| 场景 | 输入 | 输出 | 要求 |
|------|------|------|------|
| 简单链接 | `[[note]]` | `[note](./note.md)` | 链接可点击 |
| 带显示文本 | `[[note\|text]]` | `[text](./note.md)` | 显示文本正确 |
| 带锚点 | `[[note#heading]]` | `[note](./note.md#heading)` | 锚点有效 |
| 带块引用 | `[[note#^block]]` | `[note](./note.md)` 或自定义 | 块引用处理 |

### 嵌入转换

| 场景 | 输入 | 输出 | 要求 |
|------|------|------|------|
| 图片 | `![[image.png]]` | `![image](./image.png)` | 图片可显示 |
| 图片带尺寸 | `![[image.png\|300]]` | `<img src="./image.png" width="300">` 或 MDX | 尺寸保留 |
| 笔记嵌入 | `![[other-note]]` | 链接或内容嵌入 | 可配置处理 |

### Frontmatter 转换

| 字段 | 处理方式 |
|------|----------|
| `title` | 保留 |
| `date` | 保留，格式标准化 |
| `tags` | 保留，格式转换 |
| `aliases` | 移除或转换为 `title` 备选 |
| Obsidian 特有字段 | 移除或可配置映射 |

## 完成 定义

每个需求/功能应满足：

- [ ] **文档完整**：设计文档和测试记录在 `.meta/<milestone>/<feat-id>/`
- [ ] **单元测试**：领域层和应用层有对应测试
- [ ] **手动验证**：转换结果可在 fumadocs 中正确渲染
- [ ] **无回归**：现有功能不受影响
- [ ] **更新规范**：如有跨需求约束变化，更新 `.spec/`

## 常见错误

### 转换相关

1. **链接断裂**：转换后的链接路径错误
   - 原因：未正确处理相对路径
   - 解决：统一路径解析逻辑，增加测试用例

2. **编码问题**：非 ASCII 字符乱码
   - 原因：未正确处理文件编码
   - 解决：明确使用 UTF-8，处理 BOM

3. **Frontmatter 丢失**：重要元数据未保留
   - 原因：字段映射配置不完整
   - 解决：检查字段映射，添加警告日志

### 架构相关

1. **领域逻辑泄漏**：转换规则写在 CLI 层
   - 解决：将规则移至 Domain 层

2. **过度依赖基础设施**：Domain 层直接使用 `fs`
   - 解决：通过接口抽象，在 Infrastructure 层实现

3. **转换器耦合**：转换器之间相互依赖
   - 解决：每个转换器独立，通过管道组合
